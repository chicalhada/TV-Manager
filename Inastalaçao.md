TV-MANAGER – Guia de Instalação (Texto Único)
O TV-MANAGER é uma aplicação web baseada em Flask para gestão de conteúdos multimédia em ecrãs/televisores, com autenticação JWT, sincronização em tempo real via Socket.IO e arquitetura Master/Child. Para instalar, comece por garantir que tem Python 3.8 ou superior instalado, bem como o pip. Obtenha o código-fonte (via git clone ou extraindo o ZIP), abra o terminal na raiz do projeto e crie um ambiente virtual com python -m venv venv, ativando-o com venv\Scripts\activate no Windows ou source venv/bin/activate no Linux/macOS. Com o ambiente ativo, instale as dependências exatas usando pip install -r requirements.txt, que incluem Flask 2.3.3, flask-cors, flask-socketio, python-socketio, bcrypt, PyJWT e Werkzeug. Para segurança em produção, recomenda-se substituir a chave JWT fixa tvmanager_secret_key_2026_secure_32bytes por uma variável de ambiente; crie um ficheiro .env com JWT_SECRET=chave_aleatoria e edite o app.py para ler essa variável com os.environ.get. De seguida, execute python app.py – a base de dados tvmmanager.db será criada automaticamente com todas as tabelas (users, child_sites, media, playlists, playlist_items, tv_playlist_assignments, schedule) graças à chamada init_db() no arranque. O servidor ficará disponível em http://localhost:5000 (ou no IP da máquina) em modo debug. No primeiro acesso, não existem utilizadores; a aplicação permite o registo do primeiro administrador através da interface web (página de login), mas após esse registo, novos registos são bloqueados – a gestão de utilizadores passa a ser feita por administradores via API /api/users. Caso prefira, pode criar um script Python simples que use add_user do db.py para inserir um administrador diretamente na BD. Para configurar uma TV (Child), sirva os ficheiros da pasta child_website com um servidor HTTP simples (python -m http.server 8080), edite o tv_script.js para apontar para o endereço do Master e aceda a http://IP_DA_TV:8080/tv – a TV registar-se-á via WebSocket e receberá as playlists em tempo real. Em caso de problemas, verifique se a porta 5000 está livre (altere-a no app.py se necessário), instale o eventlet para suporte WebSocket (pip install eventlet) e confirme que o CORS está ativo (já está). Para produção, desative o debug (debug=False), use Gunicorn com worker eventlet (gunicorn -k eventlet -w 1 app:app) e configure um proxy reverso Nginx, além de proteger a pasta uploads/ com permissões restritas. Este guia cobre todo o processo de instalação e configuração inicial, permitindo-lhe ter o sistema operacional em poucos minutos.
